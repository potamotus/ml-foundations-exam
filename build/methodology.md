# Методология авторинга веб-тренажёра по теоретическому экзамену ML

Единый авторитетный гайд для авторов контента. Тренажёр имеет 4 режима:
**Теория** (карточки) · **Тест** (MCQ) · **Разбор формул** (открытые вопросы как в реальной КР, с эталонным ответом) · **Шпаргалка** (формулы + символы).

Привязка к реальным экзаменам: `txt/_KR2021.txt` и `txt/_KR2020_midterm.txt`.

---

## 0. Главный вывод из анализа экзамена

Реальная КР **почти не проверяет узнавание**. Она проверяет **порождение и объяснение**: записать формулу из головы, объяснить каждый символ, сказать «почему это, а не то», найти все ошибки в псевдокоде, разрешить кажущийся парадокс, посчитать метрику руками. Поэтому **всё содержание тренажёра должно заставлять студента ПРОИЗВЕСТИ ответ, а не выбрать или вспомнить готовый**.

Три педагогических столпа, на которых стоит весь гайд:
1. **Интуиция вперёд формулы.** Формула — компактная запись уже понятой идеи, а не источник идеи.
2. **Каждый символ обязан иметь имя, смысл и область значений.** Половина «непонимания» — это нераспарсенная нотация.
3. **Понимание = умение предсказать поведение, не запуская код** (предельные случаи, монотонности, «что сломается, если...»).

## 1. ШАБЛОН карточки теории (режим «Теория»)

Каждое крупное понятие оформляется по каркасу ниже. **Все секции обязательны; пустая секция = дыра в объяснении.** Карточка двусторонняя: **лицо = задание на порождение, оборот = эталон + самопроверка**. Лицо НИКОГДА не содержит формулу целиком, если задача — её воспроизвести (максимум: название и контекст).

```
ПОНЯТИЕ: <название>
ТЕМА / СВЯЗИ: <одна из тем курса; мосты к соседним понятиям через [[...]]>

1. ИНТУИЦИЯ (без формул)
   Аналогия / картинка / «на пальцах», 2–4 предложения.
   Что вообще хотим и почему наивное решение не годится.

2. ФОРМУЛА
   <каноническая запись>
   Разбор символов: вход | обучаемые параметры | гиперпараметры |
   выход | по чему сумма/среднее | область значений каждого члена.

3. ВЫВОД (скелет 3–5 шагов)
   лосс/условие → производная или приравнивание к нулю → шаг/решение →
   чтение результата словами. Не полный занудный вывод и не «ответ с неба».

4. ПОЧЕМУ ИМЕННО ТАК (обоснование лосса/критерия/знака)
   Что штрафует | из какого допущения выводится (MLE/MAP) |
   поведение на выбросах | выпуклость/дифференцируемость |
   почему этот знак/множитель/нормировка, а не другой.

5. СВЯЗИ И ИНТЕРПРЕТАЦИИ
   Геометрическая: ...
   Вероятностная: ... (если есть)
   Мосты к соседним темам (минимум один): ...

6. ТИПИЧНЫЕ ОШИБКИ → ВОПРОСЫ-УДАРНИКИ
   Миф 1 → вопрос, который его ломает.
   Миф 2 → ...
   (минимум 2 заблуждения на понятие; заблуждение должно быть СОБЛАЗНИТЕЛЬНЫМ)

7. ЧИСЛОВОЙ МИКРОПРИМЕР
   2–4 точки, считается руками на бумаге.

8. ОБОРОТ: ЭТАЛОН + ЧЕК-ЛИСТ САМОПРОВЕРКИ
   Полный образцовый ответ + 3–4 пункта «[✓ записал формулу из головы]
   [✓ назвал каждый символ] [✓ предсказал крайний случай] ...».
   По чек-листу студент сам ставит «вспомнил/не вспомнил» → калибровка + интервалка.
```

**Форма «интуиция → формула → вывод → почему → связи → типичные ошибки»** — это секции 1→2→3→4→5→6. Числовой пример и оборот с самопроверкой обслуживают всю карточку.

**Триплет карточек на каждую формулу** (последовательность строго в этом порядке):
1. **Воспроизвести целиком** — «Запиши с нуля формулу X». Лицо без формулы.
2. **Объяснить каждый символ** — только ПОСЛЕ того, как студент умеет записать формулу. Объяснять части того, что не можешь воспроизвести, бесполезно.
3. **Вывести с нуля** — worked → faded (2–3 ключевых шага заменены на `____`) → solo («покажи, что...», путь не подсказан; подсказка под спойлером, её использование сокращает интервал повторения).

**Глаголы заданий (только эти):** запиши, выведи, сформулируй, восстанови, объясни своими словами, найди ошибку, что сломается если, почему это а не то, нарисуй. **Запрещены** «выбери» и «верно ли, что» без обоснования. Любой вопрос с ответом «да/нет» дополняется «...почему?».

**Concreteness fading внутри понятия:** число → формула → смысл. Сначала «посчитай отступ для w=(1,−1), x=(2,3), y=+1», потом «запиши отступ через ⟨w,x⟩ и y», потом «что геометрически означает margin». Не наоборот.

## 2. Правила MCQ (режим «Тест»)

MCQ в теоретическом тренажёре допустим **только** как инструмент проверки ПОНИМАНИЯ, а не узнавания. Если на вопрос можно ответить, просто вспомнив определение, — переписать.

**Как писать вопрос (stem):**
- Вопрос проверяет **различение близких сущностей**, где живут заблуждения: precision vs recall, L1 vs L2, параметр vs гиперпараметр, потеря vs метрика, GD vs SGD, bias vs variance, генеративное vs дискриминативное, аналитика vs итерация.
- Предпочитать формат **«выбери и обоснуй, почему остальные неверны»** — это превращает узнавание в порождение.
- Лучшие стволы вопроса: «Что произойдёт, если...», «Какой метод/метрика тут уместен и почему», «Какое утверждение НЕВЕРНО и почему», «Что предскажет модель в крайнем случае X».
- Избегать вопросов на дословную формулировку из конспекта.

**Как писать дистракторы:**
- Каждый дистрактор — это **конкретное распространённое заблуждение**, а не случайный мусор. Соблазнительный неверный ответ создаёт desirable difficulty; абсурдный — нет.
- Минимум один дистрактор-ловушка на типичный миф темы: «margin = вероятность», «больше признаков = всегда лучше», «AUC<0.5 = плохая модель навсегда», «accuracy 99% = отличная модель», «L1 и L2 почти одно и то же», «градиент показывает к минимуму».
- Дистракторы должны быть однородны по длине и стилю с верным ответом (не выдавать ответ длиной/детальностью).

**К КАЖДОМУ варианту — объяснение (обязательно):**
- Верный вариант: **почему верно** — через причинность и связь с формулой, а не «потому что так».
- Каждый неверный: **какое именно заблуждение он эксплуатирует и чем оно опровергается** (механизм, предельный случай, контрпример).

Пример скелета:
```
ВОПРОС: При сильном дисбалансе классов (1% положительных) модель показывает
        accuracy = 0.99. Что из этого корректно следует?
A) Модель отличная.        → НЕВЕРНО: тривиальный «всегда отрицательный»
                              даёт те же 0.99, ничего не решая. Accuracy
                              чувствительна к балансу.
B) Нужно смотреть на recall/F1/PR-AUC.  → ВЕРНО: эти метрики не маскируют
                              провал на редком классе; recall ловит долю
                              пойманных положительных.
C) AUC тоже будет ≈0.99.    → НЕВЕРНО: AUC про ранжирование, не про accuracy;
                              не следует автоматически.
D) Достаточно понизить порог. → НЕВЕРНО: порог сдвигает trade-off P↔R, но
                              сам по себе не делает оценку качества честной.
```

## 3. Архетипы ОТКРЫТЫХ вопросов (режим «Разбор формул»)

Полный каталог из анализа КР 2020/2021. Калибровка: 2–3 балла за блок, 3–4 подпункта по нарастающей «вспомни → объясни → примени/разреши парадокс». Для каждого архетипа: шаблон формулировки + что ОБЯЗАН содержать эталонный ответ.

### Архетип 1. Объясни смысл величины (знак и модуль)
- **Шаблон:** «В формуле участвует [величина: отступ yᵢ⟨w,xᵢ⟩ / ⟨w,x⟩ / значение лосса]. Какой у неё смысл, что означают её знак и абсолютное значение?»
- **Эталон содержит:** знак отступа = правильность (>0 верно, <0 ошибка); модуль = уверенность/удалённость от разделяющей гиперплоскости; связку (большой «+» = уверенно и верно; большой «−» по модулю = уверенно, но НЕВЕРНО, грубейшая ошибка; около нуля = на границе, модель сомневается); связь с лоссом (отступ → +∞ ⇒ loss → 0).
- **Ловит на:** путают знак и модуль местами; забывают про уверенность; не отмечают, что большой отрицательный отступ — худший случай.

### Архетип 2. Когда величина минимальна/максимальна
- **Шаблон:** «Каким должен быть отступ, чтобы log(1+exp(−yᵢ⟨w,xᵢ⟩)) был минимальным?»
- **Эталон содержит:** отступ → +∞; тогда exp(−отступ) → 0, log(1+0) = 0; loss убывает по отступу (монотонность); важная деталь — нуля никогда не достигает точно.
- **Ловит на:** ответ «отступ = 0» или «положительный» без предела и подстановки.

### Архетип 3. Запиши и объясни формулу — разбери ВСЕ компоненты
- **Шаблон:** «Запишите формулу [шага GD и SGD / вероятности класса в логрегрессии / решающего правила]. Объясните все части.» Часто требование «общий случай, не для конкретной модели».
- **Подтипы:** 3a вероятность класса P(y=+1|x)=σ(⟨w,x⟩)=1/(1+exp(−⟨w,x⟩)); 3b шаг GD wᵗ=wᵗ⁻¹−ηₜ∇Q(wᵗ⁻¹); 3c шаг SGD — то же, но ∇ по одному объекту/мини-батчу.
- **Эталон содержит:** саму формулу + расшифровку КАЖДОГО символа + смысл знаков/операций (особенно минус в GD, нормировку сигмоиды) + области значений. Для пары GD/SGD — обязательно граница перехода (вся выборка vs один объект). Для сигмоиды — что подаётся внутрь и связь ⟨w,x⟩ = log(P₊/P₋).
- **Ловит на:** формула без объяснения минуса; путают η с коэффициентом регуляризации; в сигмоиде забывают, что внутри; не указывают области значений; пишут шаг как «+∇Q».

### Архетип 4. Сравни два метода и объясни различие
- **Шаблон:** «Чем отличаются [аналитика весов vs GD / GD vs SGD]? Чем будут отличаться результаты/наборы весов?»
- **Эталон содержит:** механизм различия И второй слой — совпадут ли результаты. Аналитика w=(XᵀX)⁻¹Xᵀy: точное решение за шаг, но обращение матрицы дорого/невозможно при большой размерности/вырожденности; GD итеративный, дешевле на больших данных; **ключевой пойнт: при единственном минимуме оба сходятся к одному решению**. GD vs SGD: вся выборка (точное направление, дорогой шаг) vs один объект (шумное направление, дешёвый шаг, масштабируемость, online, выскакивает из плохих точек).
- **Ловит на:** называют отличие в скорости, но не отвечают «совпадут ли веса»; для GD/SGD не упоминают шум/дисперсию шага.

### Архетип 5. Разреши кажущийся парадокс
- **Шаблон:** «Получаем противоречие: [A влечёт ожидание X, но наблюдается не-X]. Как объясните?»
- **Канон:** kNN без обучаемых параметров, но граница сложнее линейной.
- **Эталон содержит:** разведение «обучаемые параметры» и «сложность/гибкость/ёмкость»; kNN непараметрический — «запоминает» выборку, она и есть модель; сложность границы растёт с данными, а не с числом весов; линейный ограничен фиксированной ёмкостью d+1; **сложность ≠ число параметров**; связь с k (k=1 → очень сложная граница/переобучение, k=ℓ → недообучение).
- **Ловит на:** соглашаются с «противоречием» и доказывают, что kNN «на самом деле проще»; не вводят понятие непараметричности.

### Архетип 6. Найди ВСЕ ошибки в псевдокоде и объясни последствия
- **Шаблон:** «[Персонаж] записал схему: [псевдокод]. Найдите все ошибки и объясните, почему из-за них алгоритм работает не так. Ошибок как минимум три.»
- **Эталон содержит** (пример КР2021: w⁰=0; wᵗ=wᵗ⁻²+(1/t)·10·∇Q(wᵗ⁻¹); стоп при ‖wᵗ‖<1):
  - **Знак:** «+∇Q» вместо «−∇Q» → градиентный подъём, расходимость/максимизация потерь.
  - **Индекс:** wᵗ⁻² вместо wᵗ⁻¹ → опора на позапрошлую точку, рассогласование, не определено при t=1.
  - **Шаг/LR:** 10/t слишком велик на старте (10 при t=1) → перескоки/расходимость.
  - **Критерий останова:** ‖wᵗ‖<1 — норма весов не показатель сходимости; правильно ‖wᵗ−wᵗ⁻¹‖<ε или малость градиента; при w⁰=0 условие выполнено сразу → встанет на старте.
  - По КАЖДОЙ ошибке — обязательно ПОСЛЕДСТВИЕ для поведения, не только «тут не так».
- **Ловит на:** находят 1–2 и останавливаются («минимум три» — ловушка на полноту); указывают ошибку без эффекта; пропускают ошибку в критерии останова.

### Архетип 7. Посчитай метрику вручную: ROC и AUC
- **Шаблон:** «Нарисуйте ROC-кривую и посчитайте AUC-ROC» (дана таблица yᵢ + b(xᵢ)).
- **Эталон содержит:** сортировку по убыванию b(x); пошаговый перебор порогов; на каждом — TPR=TP/(TP+FN), FPR=FP/(FP+TN); движение по сетке (вверх на 1/P за «+», вправо на 1/N за «−»); AUC = площадь под ломаной (или доля правильно упорядоченных пар +/−); корректная нормировка осей; ничьи дают диагональный шаг.
- **Ловит на:** путают оси TPR/FPR; неверно считают P и N; забывают про равные вероятности; неаккуратная площадь.

### Архетип 8. Посчитай precision/recall при заданном пороге
- **Шаблон:** «Посчитайте точность и полноту при пороге t=[значение]» / «Бинаризуйте и посчитайте precision, recall, F1.»
- **Эталон содержит:** правило бинаризации (b(x)≥t → +1); явный подсчёт TP, FP, FN, TN; Precision=TP/(TP+FP); Recall=TP/(TP+FN); F1=2PR/(P+R); аккуратная обработка границы (≥ или >).
- **Ловит на:** путают FP/FN; путают знаменатели; неверно относят пограничный объект; ошибки деления.

### Архетип 9. Достижимы ли target recall при target precision (оптимизация по порогу)
- **Шаблон:** «Можно ли достичь полноты ≥X% при точности ≥Y%? Если да — при каком пороге?»
- **Эталон содержит:** понимание trade-off P↔R; перебор порогов по уникальным значениям вероятностей; для каждого пара (P,R); явную проверку условия; ответ «да + порог» или «нет» с обоснованием; частный случай recall=100% (минимальный безопасный порог).
- **Ловит на:** проверяют один порог вместо перебора; не понимают связь порог→recall; не находят порог для recall=100%.

### Архетип 10. Что значит свойство метрики и как его обойти
- **Шаблон:** «Accuracy чувствительна к балансу классов. Что это значит? Что сделать для корректной оценки? Аргументируйте.»
- **Эталон содержит:** объяснение проблемы (тривиальный «мажоритарный» даёт высокую accuracy при дисбалансе); альтернативы (precision/recall/F1, ROC-AUC, balanced accuracy, PR-AUC); **аргументацию выбора** (почему именно она устойчива); числовой контрпример (99% одного класса → accuracy 99% у бесполезной модели).
- **Ловит на:** называют проблему без альтернативы; альтернатива без аргументации (экзамен требует «аргументируйте»); путают, какие метрики устойчивы.

### Архетип 11. Свойства/инварианты AUC при преобразовании алгоритма
- **Шаблон:** «AUC b(x)=0.1, постройте алгоритм лучше» / «c(x)=b(x)/3, чему равен AUC?»
- **Эталон содержит:** ключевую интуицию — **AUC зависит только от порядка (ранжирования), не от значений**; монотонное преобразование (деление на 3) ранжирование не меняет → AUC тот же; AUC<0.5 → алгоритм системно переворачивает классы, c(x)=1−b(x) даёт AUC=1−0.1=0.9 (инверсия осей).
- **Ловит на:** пересчитывают ROC численно вместо рассуждения об инвариантности; не знают свойство AUC→1−AUC при инверсии.

### Архетип 12. Связь β в F_β с приоритетом точности/полноты
- **Шаблон:** «Если точность в N раз важнее полноты, чему равно β? Посчитайте F_β.»
- **Эталон содержит:** F_β=(1+β²)PR/(β²P+R); правило β<1 → важнее точность, β>1 → важнее полнота; корректный подбор β; аккуратную подстановку.
- **Ловит на:** путают направление β; неверно переводят «в N раз важнее» в β; путают с F1.

### Архетип 13. Докажи геометрическое свойство метода (kNN)
- **Шаблон:** «Покажите, что граница 1-NN между двумя точками — прямая.»
- **Эталон содержит:** запись равенства/неравенства расстояний, раскрытие квадратов, **сокращение квадратичных членов**, вывод линейного неравенства; альтернативно — серединный перпендикуляр (ГМТ равноудалённых точек).
- **Ловит на:** не замечают, что квадраты координат сокращаются.

### Архетип 14. Найди границы классов для вариантов kNN
- **Шаблон:** «Постройте границы 1-NN / 2-NN / взвешенного / 3-NN на одномерной выборке.»
- **Эталон содержит:** явные границы для каждого варианта; правило разрешения ничьих (меньшая метка); корректный учёт весов.

### Архетип 15. Гиперпараметры метода и их влияние на пере-/недообучение
- **Шаблон:** «Перечислите числовые гиперпараметры kNN и как они влияют.»
- **Эталон содержит:** k — единственный числовой гиперпараметр; k=1 → переобучение, k=ℓ → недообучение; механизм (усреднение/сглаживание границы).
- **Ловит на:** полноте перечня.

### Архетип 16. Сколько параметров vs гиперпараметров настраивать
- **Шаблон:** линейные модели ± Lasso ± полиномиальные признаки.
- **Эталон содержит:** разведение параметров (веса, зависят от числа признаков, +1 за константу) и гиперпараметров (коэффициент регуляризации — только при регуляризации, =0 без неё).
- **Ловит на:** путанице параметр/гиперпараметр и учёте константного признака.

### Архетип 17. Сколько наборов весов дают идеальную классификацию
- **Шаблон:** по таблице определить мощность множества решений (0/1/2/∞) и привести примеры/условие на веса.
- **Эталон содержит:** вывод условия на (w₀,w₁,…); доказательство, что решений бесконечно (линейная разделимость с зазором) или конечно; явные наборы.
- **Ловит на:** скачке от «посчитать» к «доказать мощность множества».

### Архетип 18. Выбор признака для первого узла дерева
- **Шаблон:** по таблице выбрать признак первого сплита (энтропия для классификации / дисперсия для регрессии).
- **Эталон содержит:** формулы H(R) и Q=(|R_l|/|R_m|)H(R_l)+(|R_r|/|R_m|)H(R_r)→min; подсчёт для КАЖДОГО кандидата; сравнение и выбор минимума; обоснование весов (иначе разбиение «1 объект vs остальные» получит нечестно низкую сумму).
- **Ловит на:** нужно посчитать ОБЕ ветви и взвесить (частая ошибка — одна ветвь).

### Архетип 19. Взвешенная kNN-регрессия по графику
- **Шаблон:** предсказать значение в точке взвешенным k-NN с весами wᵢ=(k−i+1)/k.
- **Эталон содержит:** a(x)=Σwᵢyᵢ/Σwᵢ; выбор k ближайших; присвоение весов по рангу (ближайший — наибольший вес); аккуратное усреднение.
- **Ловит на:** присвоении весов по рангу и арифметике.

**Сквозные принципы проверяющего (отличают полный ответ от половинчатого):**
1. Формула без расшифровки символов = неполный ответ. Всегда: записать → назвать каждый символ → смысл → область значений и роль знаков.
2. Знаки и пределы — главные крючки: минус в GD, направление β, отступ→+∞, AUC<0.5→инверсия.
3. «Найди все» / «минимум три» — проверка ПОЛНОТЫ; недобор штрафуется.
4. Парадоксы решаются введением правильного понятия (непараметричность, инвариантность AUC, ёмкость ≠ число параметров), а не пересчётом.
5. «Аргументируйте» требует явного обоснования, не только названия альтернативы.
6. В расчётах показывать промежуточные TP/FP/FN/TN, P и N, пороги; голый числовой ответ недооценивается.
7. Сравнительные вопросы требуют второго слоя: «совпадут ли результаты / изменится ли метрика».

## 4. Формат записи в ШПАРГАЛКЕ

Шпаргалка — это не пересказ теории, а **плотный справочник формул и символов**. Один блок на формулу, строго по шаблону:

```
[Название] · [Тема]

ФОРМУЛА:  <каноническая запись>

СИМВОЛЫ:
  <символ> — <что это: вход/обучаемый параметр/гиперпараметр/выход> [область значений]
  <символ> — ...
  (расписать КАЖДЫЙ символ, без пропусков; индекс суммы/среднего — тоже символ)

КОГДА / ЗАЧЕМ:  <одна строка: в какой задаче применяется и что даёт;
                ключевой знак/предел/инвариант, если он — крючок>
```

Пример:
```
Шаг градиентного спуска · Оптимизация

ФОРМУЛА:  wᵗ = wᵗ⁻¹ − ηₜ · ∇Q(wᵗ⁻¹)

СИМВОЛЫ:
  wᵗ      — приближение весов на шаге t (обучаемые параметры)
  wᵗ⁻¹    — предыдущее приближение
  ηₜ      — шаг/learning rate (гиперпараметр, может убывать с t), >0
  ∇Q      — градиент функционала по ВСЕЙ выборке
  «−»     — движемся ПРОТИВ градиента, в сторону убывания Q

КОГДА / ЗАЧЕМ:  итеративная минимизация лосса, когда нет/дорого замкнутое
                решение; крючок — минус (без него градиентный подъём).
```

Правила: каждый символ обязательно с типом (вход/параметр/гиперпараметр/выход) и областью значений; знаки и пределы, которые являются «крючками» экзамена, выносить в строку «когда/зачем»; формулы группировать по темам; парные формулы (GD/SGD, precision/recall, L1/L2) ставить рядом для различения.

## 5. Акценты по темам, припоминание и перемешивание

**Темы курса** (12): kNN; линейная регрессия и MSE; градиентный спуск и SGD; регуляризация L1/L2; bias–variance; логистическая регрессия и отступы; SVM; метрики классификации (precision/recall/ROC/AUC); решающие деревья; бэггинг/случайный лес; бустинг; кластеризация и снижение размерности.

**Акценты (куда экзамен бьёт сильнее всего — отрабатывать в первую очередь):**
- **Отступ и логрегрессия:** смысл знака/модуля, вывод log-loss из log-odds, почему log-loss, а не MSE/accuracy.
- **Градиентный спуск:** знак минус, learning rate, критерий останова, ошибки в псевдокоде, GD vs SGD, совпадение весов с аналитикой.
- **Метрики:** ROC/AUC руками, инвариантность AUC к монотонным преобразованиям, чувствительность accuracy к дисбалансу с аргументацией альтернативы, trade-off precision↔recall, β в F_β.
- **Деревья:** взвешенный критерий Q (обе ветви!), энтропия/дисперсия, выбор первого сплита.
- **kNN:** парадокс непараметричности, граница 1-NN как прямая, влияние k, нормировка.
- **Сквозные мосты (подсвечивать всегда):** линейная+гаусс=МНК; логрегрессия=линейная+сигмоида+log-loss; регуляризация=MAP-приор (L2 гаусс, L1 лаплас); бэггинг↓variance vs бустинг↓bias; сложность ≠ число параметров.

**Предельные случаи и монотонности** (главный инструмент проверки понимания — закладывать в каждую тему): k→1 и k→N; λ→0 и λ→∞; C→0 и C→∞; глубина→∞; η слишком велик/мал; batch=1 vs batch=N; число деревьев в бэггинге (не переобучает) vs бустинге (переобучает).

**Припоминание (retrieval practice):** запрещён формат «текст для прочтения». Каждая единица провоцирует активный ответ ДО показа решения. Оборот всегда содержит чек-лист самопроверки → студент сам ставит «не вспомнил / с трудом / уверенно».

**Worked → faded → solo:** новую формулу/вывод давать сначала полностью разобранной, потом с пропусками, потом самостоятельно. Solo не давать, пока faded не пройдена уверенно дважды.

**Интервалка по самооценке:** 3 кнопки «не вспомнил / с трудом / уверенно» (SM-2-подобно). «Не вспомнил» возвращает карточку в тот же сеанс; «уверенно» растягивает интервал; использованная подсказка трактуется как «с трудом».

**Перемешивание (interleaving):** в сеанс повторения подмешивать карточки из РАЗНЫХ освоенных тем — чтобы студент учился ВЫБИРАТЬ метод/метрику, а не работать по инерции одной темы. Обязателен тип карточки «какой метод/метрика тут уместен и почему». **Новый материал вводится блоком** (worked→faded→solo); в перемешанный пул уходит только освоенное. Interleaving — для закрепления, не для первого знакомства.
